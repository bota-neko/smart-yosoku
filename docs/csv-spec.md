# CSV取込み仕様

過去実績（1年以上）を Excel/CSV から取り込むための仕様。ロジックは `src/lib/csv.ts`（純粋・テスト済み）。

## 最小要件
以下の4種があれば取込可能:
- **日付**（YYYY-MM-DD、YYYY/MM/DD も可）
- **拠点名**
- **予測対象名**（商品/メニュー等）
- **実績値**（販売数など、いずれか1つ以上）

## 対応列（自動判定の同義語）
| フィールド | 認識される見出し例 |
|---|---|
| 日付 | 日付, 年月日, date |
| 拠点 | 拠点, 店舗, 工場, store |
| 予測対象 | 商品, メニュー, 品目, item, product |
| 販売数 | 販売数, 売上数, sold |
| 納品数 | 納品数, delivered |
| 製造数 | 製造数, 生産, produced |
| 発注数 | 発注数, ordered |
| 在庫数 | 在庫数, stock |
| 返品数 | 返品数, returns |
| 廃棄数 | 廃棄数, ロス, waste |
| 欠品数 | 欠品数, stockout |
| 来客数 | 来客数, 客数, visitors |
| 予約数 | 予約数, reservations |
| 売上金額 | 売上金額, 金額, amount |
| 備考 | 備考, メモ, note |

見出しが一致しない場合は **手動マッピング** で列を割当。

## 取込みフロー
1. **サンプルCSVダウンロード**（`sampleCsv()`）。
2. アップロード（MIME/拡張子/サイズ検証。既定上限5MB）。
3. `parseCsv` でパース（クオート・カンマ・改行対応、空行スキップ）。
4. `autoMapColumns` で列自動判定 → 必要なら手動修正。
5. `mapAndValidate` で行検証（日付形式・必須項目・実績値有無）。
6. `buildPreview` で **取込前プレビュー**（総数/有効数/エラー行/重複キー）。
7. 重複 `(日付, 拠点, 対象)` の処理を選択: **新規追加 / 上書き / スキップ**。
8. 取込実行 → `csv_imports` / `csv_import_rows` に履歴保存。
9. **ロールバック**: import 単位で挿入行を削除して取消可能な設計。

## 0 と 未入力 の区別
空セルは `null`（未入力）、`0` はゼロ実績として区別して保存する（`nullableNumber`）。

## エラー処理
- 不正な行はスキップせず **エラー行として提示**（行番号・理由）。
- 数千行でも異常終了しない（ストリーミング的にプレビュー集計）。
- 数値の桁区切りカンマ（`1,200`）は自動除去。
